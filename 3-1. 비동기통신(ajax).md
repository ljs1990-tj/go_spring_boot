# 🍽️ 3단계: 화려한 홀 서빙 기술 (Vue.js 데이터 바인딩과 Ajax 완벽 해부)

주방(Spring Boot 백엔드) 세팅을 완벽하게 마치신 것을 진심으로 축하합니다! 이제 주방장은 손님의 주문을 받아 맛있는 요리(JSON 데이터)를 만들어낼 준비가 모두 끝났습니다.

하지만 요리가 아무리 맛있어도, 손님 테이블(웹 브라우저)에 서빙되지 않으면 아무 소용이 없겠죠? 이번 3단계에서는 손님들이 앉아있는 홀(JSP 화면)을 관리하는 **최첨단 지배인(Vue.js)**과 주방과 홀을 빛의 속도로 오가는 **투명 인간 점원(Ajax)**에 대해 아주 깊이 있게 알아보겠습니다.

---

## 1. 과거의 식당(전통적 웹) vs 현대의 식당(Vue.js + Ajax)

먼저 우리가 왜 굳이 Vue.js와 Ajax라는 기술을 써야 하는지, 그 근본적인 이유부터 이해해야 합니다.

* **과거의 식당 (새로고침 방식):** 손님이 메뉴를 하나 추가할 때마다 식당 불이 전부 꺼지고 손님을 식당 밖으로 내쫓습니다. 그리고 메뉴 하나가 추가된 똑같은 식당을 새로 지어서 손님을 다시 입장시킵니다. 웹 페이지에서 버튼을 누를 때마다 **화면 전체가 하얗게 변하며 깜빡이는(새로고침) 현상**이 바로 이것입니다. 서버(주방)가 매번 화면(JSP) 전체를 새로 그려서 통째로 보내주었기 때문이죠. 엄청난 낭비입니다!

* **현대의 식당 (비동기 통신 방식 - SPA 개념):**
  손님은 자리에 가만히 앉아 있습니다. 식당 불도 꺼지지 않습니다. 손님이 추가 주문을 하면, 투명 인간 점원(Ajax)이 몰래 주방에 가서 **'요리(데이터)'**만 접시에 담아옵니다. 그리고 최첨단 지배인(Vue.js)이 손님 테이블의 텅 빈 접시를 요리가 담긴 접시로 스르륵 바꿔치기합니다. **화면 깜빡임 없이 데이터만 부드럽게 바뀌는 현대적인 웹 서비스**의 원리입니다.

## 2. 최첨단 홀 지배인, Vue.js의 뇌 구조 파헤치기

여러분이 작성하신 `<script>` 태그 안의 `Vue.createApp({ ... })` 코드는 바로 이 최첨단 지배인을 채용하고 뇌를 세팅하는 과정입니다. 지배인의 뇌는 크게 3개의 핵심 구역으로 나뉘어 있습니다.

### 📝 1구역: `data()` - 지배인의 마법 메모장 (상태 관리와 반응성)
`data()` 구역은 화면에 보여줄 모든 변수(데이터)를 기록해 두는 지배인의 전용 메모장입니다.
* **핵심 원리 (반응성 - Reactivity):** Vue.js가 위대한 이유는 이 메모장에 **'반응성'**이라는 마법이 걸려있기 때문입니다. 지배인이 메모장에 `welcomeMessage = "안녕!"` 이라고 적어두고 화면(HTML)에 그 글자를 띄워두었다고 가정해 봅시다. 
* 만약 중간에 지배인이 메모장의 글자를 `welcomeMessage = "반가워!"` 라고 지우고 새로 쓰면 어떻게 될까요? 여러분이 HTML 코드를 직접 건드리지 않아도, **메모장의 값이 변하는 그 즉시! 화면에 떠 있던 글자가 스스로 "반가워!"로 스르륵 변신**합니다. 우리는 이것을 **'데이터 바인딩(Data Binding)'**이라고 부릅니다.
* **사용 예시:** 빈 게시판 목록을 보여주기 위해 처음에는 메모장에 `boardList: []` 라고 텅 빈 배열(접시)을 준비해 둡니다.

### 🏃‍♂️ 2구역: `methods` - 지배인의 행동 지침서 (이벤트 처리)
메모장이 명사(데이터)라면, `methods`는 동사(행동)입니다. 손님이 어떤 행동(클릭, 키보드 입력 등)을 했을 때 지배인이 어떻게 대처해야 하는지 매뉴얼을 적어두는 곳입니다.
* **사용 예시:** "최신 목록 불러오기" 버튼을 HTML에 만들어두고, 버튼에 `@click="fnList"` 라는 이름표를 달아줍니다. 그러면 손님이 버튼을 클릭할 때마다 지배인은 매뉴얼을 펴서 `fnList: function() { ... }` 안에 적힌 행동을 그대로 수행합니다.

### ⏰ 3구역: `mounted()` - 식당 오픈 직전의 최종 점검 (라이프사이클)
식당 문을 열기(화면을 브라우저에 띄우기) 직전에, 딱 한 번 자동으로 실행해야 하는 일들을 적어두는 곳입니다.
* **사용 예시:** 손님이 식당에 들어오자마자 텅 빈 게시판을 보면 당황하겠죠? 화면이 준비되자마자 지배인이 알아서 `methods`에 있는 `fnList` 행동을 강제로 한 번 실행하도록 지시합니다. (초기 데이터 불러오기)

## 3. 투명 인간 점원: jQuery `$.ajax` 해부학

지배인(Vue)이 주문을 확인했다면, 이제 주방(Spring Boot)으로 점원을 보내야 합니다. 우리가 사용할 투명 인간 점원의 이름은 **jQuery Ajax** 입니다. `methods` 안에서 주로 활동하며, 출발 전 5가지 필수 지시사항을 받아야만 움직입니다.

1. **`url` (목적지):** 주방의 수많은 창구 중 어디로 갈 것인가? (예: Controller에서 약속한 `@PostMapping("/api/board/list")` 주소를 적어줍니다.)
2. **`type` (전달 방식):** "주문서를 어떻게 들고 갈까?" (우리는 백엔드에서 `POST`로 받기로 했으므로 "POST"라고 적습니다.)
3. **`dataType` (기대하는 요리 형태):** "주방에서 요리를 받아올 때 어떤 그릇에 담아달라고 할까?" (우리는 무조건 "json" 형태를 요구합니다. 자바스크립트가 가장 좋아하는 객체 형태입니다.)
4. **`data` (전달할 파라미터):** 손님의 세부 요구사항입니다. "3번 게시글 보여줘" 라면 `param = { id: 3 }` 처럼 만들어서 들려 보냅니다.
5. **`success` (복귀 후 행동):** ★가장 중요합니다! 점원이 주방에서 갓 만든 요리(JSON 데이터)를 들고 무사히 홀로 돌아왔을 때, 이 요리를 어떻게 할 것인가를 적습니다. 보통 여기서 받아온 요리를 `data()` 메모장에 쏟아붓습니다.

## 4. 🚨 3단계에서 초보자가 100% 겪는 눈물의 에러 (트러블슈팅 가이드)

프론트엔드(Vue)와 백엔드(Spring)가 처음으로 입을 맞추는 구간입니다. 여기서 수많은 초보자가 포기하지만, 다음 두 가지만 알면 절대 무너지지 않습니다.

### ❌ 실수 1: 점원이 지배인을 잊어버리는 병 (Ajax 안에서의 `this` 상실)
* **절망적인 상황:** 주방에서 요리 데이터는 무사히 받아왔습니다. 그래서 `success: function(data)` 안에서, 마법의 메모장에 요리를 담으려고 `this.boardList = data;` 라고 썼습니다. 그런데 화면에 아무것도 안 뜨고 콘솔 창에 `this is undefined` (this가 누군지 모름!) 에러가 뜹니다.
* **원인 (자바스크립트의 함정):** 자바스크립트에서 `this`는 '나를 부른 주체'를 가리킵니다. 평소에 `this`는 홀 지배인(Vue 인스턴스)을 가리킵니다. 하지만 **Ajax 통신 안으로 깊숙이 들어가는 순간, `this`의 정체성은 지배인이 아니라 '점원(Ajax 요청 자체)'으로 뒤바뀌어 버립니다!** 점원은 메모장(`boardList`)을 가지고 있지 않으니 에러가 나는 것입니다.
* **생명줄 (해결책):** 여러분이 제공해주신 템플릿에 항상 **`let self = this;`** 라는 코드가 가장 위에 적혀있던 것을 기억하시나요? 이것이 바로 치트키입니다! 점원이 출발하기 전에, 원래 지배인의 정체성(`this`)을 `self`라는 변수에 안전하게 복사해 두는 것입니다. 그리고 Ajax 안에서는 `this` 대신 **`self.boardList = data;`** 라고 적으면 메모장에 완벽하게 기록됩니다!

### ❌ 실수 2: 백엔드 변수와 프론트엔드 변수의 대소문자 불일치
* **상황:** 데이터는 메모장에 잘 들어왔는데, HTML 화면에는 텅 빈 테이블만 생깁니다.
* **원인:** 백엔드(Java DTO)에서 글 제목을 `title` 이라고 지어놓고 보냈는데, 프론트엔드(HTML)에서는 `{{ Title }}` 이나 `{{ TITLE }}` 처럼 대소문자를 다르게 적었기 때문입니다.
* **해결책:** 컴퓨터는 바보라서 대문자 T와 소문자 t를 전혀 다른 글자로 인식합니다. 자바의 DTO 변수명과 HTML의 `{{ }}` 안의 변수명이 완벽하게 똑같은지 항상, 언제나, 복사해서 붙여넣는 습관을 들이세요!
