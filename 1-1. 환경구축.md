# 🏗️ 1단계: 식당 부지 매입과 주방 세팅 (스프링 부트 환경 구축) 

드디어 나만의 웹 서비스라는 '거대한 식당'을 개업할 시간입니다! 
하지만 맨땅에 벽돌을 하나하나 쌓아서 식당을 지으려면 수년이 걸리겠죠. 그래서 우리는 **스프링 부트(Spring Boot)**라는 아주 훌륭한 '건축 회사'의 도움을 받을 것입니다. 

이번 1단계에서는 식당의 뼈대를 세우고, 손님을 맞이할 홀(JSP)과 주방(Java)의 위치를 세팅하는 가장 기초적이고 중요한 작업을 진행합니다.

---

## 1. 프로젝트 생성하기: "어떤 식당을 지어드릴까요?"

가장 먼저 'Spring Initializr(스프링 이니셜라이저)'라는 사이트나 IDE(이클립스, 인텔리제이)의 기능을 이용해 프로젝트를 생성해야 합니다. 이는 건축 회사에 **"이런이런 자재를 써서 식당을 지어주세요"라고 주문서를 넣는 과정**입니다.

### 🛠️ 핵심 주문 내역 (개발 환경)
1. **빌드 도구 (Maven):** * **비유:** 식당에 필요한 자재(밀가루, 냉장고 등)를 알아서 사다 주고 관리해 주는 **'만능 조달청 직원'**입니다. `pom.xml`이라는 주문서에 "스프링 웹 기능 필요해!"라고 한 줄만 적으면, Maven이 인터넷에서 관련 부품을 싹 다 다운로드해 줍니다.
2. **언어 (Java):** 주방장(서버)이 사용할 요리 언어입니다.
3. **스프링 부트 버전 (3.x):** 가장 최신의 안정적인 뼈대를 사용합니다.
4. **Dependencies (의존성 - 필수 주방 기기):**
   * **Spring Web:** 인터넷을 통해 손님의 주문(HTTP 요청)을 받을 수 있게 해주는 가장 핵심적인 기능입니다.
   * **MyBatis Framework:** 나중에 배울 '냉장고(DB)'에서 재료를 쉽게 꺼내오게 해주는 마법의 국자입니다.
   * **MySQL Driver (또는 Oracle, H2):** 사용할 데이터베이스의 종류에 맞춘 전용 통신 케이블입니다.
   * **Lombok:** 겟터/셋터(Getter/Setter) 같은 귀찮은 반복 코드를 한 줄로 줄여주는 요리 보조 로봇입니다.

---

## 2. 🚨 가장 중요한 고비: JSP를 위한 특별 허가증 발급

스프링 부트는 기본적으로 '타임리프(Thymeleaf)'라는 최신식 인테리어 자재를 좋아합니다. 그래서 우리가 사용하기로 한 **JSP(Java Server Pages)**를 홀(화면)에 깔려고 하면, 처음엔 스프링 부트가 인식하지 못합니다.

이를 해결하기 위해 Maven 주문서(`pom.xml`)에 **'JSP 특별 허가증'**을 발급받아야 합니다.
* **필수 추가 부품:** `tomcat-embed-jasper`
* **이유:** 스프링 부트 안에는 '톰캣(Tomcat)'이라는 서빙 직원이 내장되어 있는데, 이 직원이 JSP 문서를 읽고 해석할 수 있도록 번역기(Jasper)를 달아주는 아주 중요한 작업입니다. 이 코드가 없으면 화면에 404 에러(페이지를 찾을 수 없음)가 뜹니다.

---

## 3. 환경 설정 (`application.properties`): 식당 내부 구조 그리기

부품이 다 모였으니, 이제 `application.properties` (또는 `application.yml`) 파일에 식당의 세부 규칙과 도면을 적어주어야 합니다. 

### 🗺️ View Resolver (화면 경로 설정)
서빙 직원(Controller)이 손님에게 메뉴판(JSP 화면)을 보여주려 할 때, 메뉴판이 식당의 어느 구석에 박혀있는지 정확한 주소를 알려주어야 합니다.

* **prefix (접두사):** `/WEB-INF/views/`
  * "우리의 모든 화면(JSP) 파일은 이 폴더 안에 숨겨둘 거야!" 라는 뜻입니다. `WEB-INF` 폴더는 외부 손님이 인터넷 주소창을 통해 강제로 뚫고 들어올 수 없는 아주 안전한 금고 역할을 합니다.
* **suffix (접미사):** `.jsp`
  * "파일 이름 뒤에는 무조건 `.jsp`가 붙어있어!" 라는 뜻입니다. 
* **효과:** 서빙 직원이 `index`라는 화면을 보여주라는 지시를 받으면, 알아서 앞뒤로 글자를 붙여서 `/WEB-INF/views/index.jsp`를 찾아가게 됩니다. 개발자는 매번 긴 주소를 칠 필요가 없어집니다!

---

## 4. 프론트엔드 템플릿: Vue.js와 jQuery를 CDN으로 부르기

이제 손님이 밥을 먹을 홀(JSP)을 꾸밀 차례입니다. 우리는 최신 기술인 **Vue.js 3**와, 통신을 도와줄 **jQuery**를 사용하기로 약속했습니다. 

복잡하게 파일을 다운로드해서 서버에 넣을 필요 없이, **CDN(Content Delivery Network)** 방식을 사용합니다.
* **CDN의 비유:** 가구를 직접 사서 식당에 들이는 게 아니라, **"가구 대여점 주소만 적어두면, 손님이 식당에 들어올 때마다 인터넷에서 실시간으로 빔프로젝터처럼 가구를 쏘아서 보여주는 방식"**입니다.
* 제공해주신 HTML/JSP 템플릿의 `<head>` 태그 안에 있는 `<script src="http...">` 코드가 바로 이 대여점 주소입니다. 이 주소만 적어두면 웹 브라우저가 알아서 Vue.js와 jQuery 기능을 화면에 입혀줍니다.

## 5. 🚨 초보자가 1단계에서 가장 흔히 겪는 에러 (트러블슈팅 가이드)

프로젝트를 만들고 처음으로 초록색 재생 버튼(Run)을 눌렀을 때, 90%의 초보자가 마주하는 에러들입니다. 당황하지 마세요!

### ❌ 에러 1: "Web server failed to start. Port 8080 was already in use."
* **상황:** 시작하자마자 빨간 글씨가 뜨며 서버가 죽어버립니다.
* **원인:** 스프링 부트의 기본 출입문 번호(포트)는 `8080`입니다. 그런데 여러분의 컴퓨터에 오라클 DB나 다른 프로그램이 이미 8080번 문을 차지하고 있어서, 스프링 부트가 문을 열지 못하고 충돌한 것입니다.
* **해결책:** `application.properties` 파일에 가서 `server.port=8081` (또는 8090 등)로 출입문 번호를 바꿔주면 깔끔하게 해결됩니다!

### ❌ 에러 2: 404 Not Found (화면이 안 떠요!)
* **상황:** 브라우저에 `localhost:8080`을 쳤는데 하얀 화면에 404 에러만 뜹니다.
* **원인:** 백엔드(자바)는 정상적으로 켜졌으나, 아까 언급한 `tomcat-embed-jasper` 번역기가 없거나, `application.properties`의 prefix 경로 오타, 혹은 실제 JSP 파일을 해당 폴더 안에 만들지 않았을 때 발생합니다. 경로의 스펠링을 다시 한번 꼼꼼히 확인하세요.
