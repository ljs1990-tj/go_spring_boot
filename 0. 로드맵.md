# 🚀 Spring Boot x Vue.js (CDN) 실전 팀 프로젝트 로드맵 

웹 개발은 거대한 **'식당'**을 개업하고 운영하는 것과 같습니다. 주방(DB와 백엔드), 홀(프론트엔드), 그리고 주문을 전달하는 점원(Ajax)이 완벽하게 호흡을 맞춰야 하죠. 이 로드맵을 따라오시면 어엿한 풀스택 식당의 주인이 되실 수 있습니다!

---

## 🏗️ 1단계: 식당 부지 매입과 주방 세팅 (환경 설정 및 기본 구조)
본격적인 요리에 앞서 기본 환경을 세팅하는 과정입니다.

* **핵심 내용:** * Spring Boot 프로젝트 생성 (Maven)
  * application.properties (또는 .yml) 데이터베이스 연결 설정
  * JSP 화면을 띄우기 위한 라이브러리 추가 및 설정
  * 템플릿(Vue 3 + jQuery)이 적용된 첫 페이지 띄워보기
* **학습 포인트:** 톰캣(Tomcat) 서버가 켜지고 내 브라우저에 `localhost:8080`이 뜨는 첫 성공의 기쁨을 맛보는 단계입니다.

## 🧑‍🍳 2단계: 백엔드 3형제와 비밀 레시피 (Spring MVC & MyBatis)
주방에서 재료를 다듬고 요리를 만드는 백엔드의 핵심 구조를 배웁니다.

* **핵심 내용:**
  * **Controller:** 주문을 받는 매니저
  * **Service:** 요리 순서를 결정하는 메인 셰프
  * **Mapper (MyBatis) & XML:** 냉장고(DB)에서 재료를 꺼내오는 조수
* **학습 포인트:** 사용자의 요청이 Controller -> Service -> Mapper -> DB로 흘러가는 '단방향 물결'의 흐름을 완벽하게 이해합니다.

## 🍽️ 3단계: 화려한 홀 서빙 기술 (Vue.js 데이터 바인딩과 Ajax)
손님(사용자)에게 보여줄 화면(JSP)을 Vue.js로 살아 움직이게 만들고, 백엔드와 통신하는 법을 배웁니다.

* **핵심 내용:**
  * Vue.js의 `data()` 변수 선언과 화면에 데이터 뿌려주기 (`{{ }}`)
  * 버튼 클릭 시 실행되는 `methods` 함수 만들기
  * **jQuery Ajax:** 프론트엔드(Vue)와 백엔드(Spring)를 연결해 주는 마법의 점원
* **학습 포인트:** 새로고침 없이 화면의 데이터만 쏙쏙 바뀌는 비동기 통신의 매력을 체감합니다.

## 🛠️ 4단계: 핵심 메뉴 개발 (게시판 CRUD 실전)
모든 웹 서비스의 뼈대가 되는 CRUD(생성, 읽기, 수정, 삭제)를 본격적으로 구현합니다.

* **핵심 내용:**
  * **C (Create):** 글쓰기 화면 만들고 DB에 Insert 하기
  * **R (Read):** 전체 목록 화면(List)과 상세 보기 화면(Detail) Select 하기
  * **U (Update) & D (Delete):** 내가 쓴 글 수정하고 삭제하기
* **학습 포인트:** 이전에 약속한 JSP Vue 템플릿을 적극 활용하여, 백엔드에서 만든 JSON 데이터를 화면의 테이블에 예쁘게 그리는 법을 집중 연습합니다.

## 🤝 5단계: 팀 프로젝트 실전 협업과 에러 대처법
이제 각자 역할을 나누어 하나의 거대한 프로젝트를 완성합니다.

* **핵심 내용:**
  * 화면 설계서(UI/UX)와 API 명세서(메뉴판) 작성의 중요성
  * 화면 담당(프론트)과 API 담당(백엔드)의 업무 분담 방법
  * 실전 배포 전 최종 테스트
* **학습 포인트:** "내 컴퓨터에선 되는데 네 컴퓨터에선 안 돼!" 현상을 방지하는 협업의 규칙을 배웁니다.

## 🚨 초보 팀 프로젝트 단골 실수 (미리보는 트러블슈팅)

### ❌ 실수 1: 프론트와 백엔드의 '변수명(Key)' 불일치
* **상황:** 백엔드(MyBatis/Java)에서는 이름을 `userName`으로 보냈는데, 프론트엔드(Vue.js)에서는 `user_name`으로 찾고 있어서 화면에 아무것도 안 뜨는 현상입니다. 
* **예방책:** 개발을 시작하기 전에 항상 **API 명세서(어떤 이름표를 달아서 데이터를 주고받을지 약속한 문서)**를 먼저 작성하고 코딩해야 합니다!

### ❌ 실수 2: Ajax 통신 시 404, 500 에러의 늪
* **상황:** 버튼을 눌렀는데 반응이 없고 콘솔 창에 빨간 줄이 가득합니다.
* **해결책:** * **404 (Not Found):** Ajax의 `url` 주소와 Spring Controller의 `@RequestMapping` 주소가 오타 없이 똑같은지 확인하세요. (점원이 길을 잃은 것입니다)
  * **500 (Internal Server Error):** 주소는 맞게 찾아갔으나 주방(서버)에서 요리 중 냄비가 터진 것입니다. 십중팔구 **SQL 쿼리문 오타**나 **Null 값 처리** 문제이므로 이클립스(IntelliJ) 콘솔 로그를 확인해야 합니다.
